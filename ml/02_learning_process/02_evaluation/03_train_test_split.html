Train/Test Split

1º - O Problema: Avaliar no mesmo dado é errado

Se treinarmos e avaliarmos um modelo usando os mesmos dados, teremos uma métrica irrealista.

O modelo pode simplesmente "decorar" os dados.

Isso leva a uma avaliação otimista e enganosa.

---

2º - Ideia do Train/Test Split

Dividimos os dados em duas partes:

  • Treino (train): usado para ajustar o modelo
  • Teste (test): usado para avaliar o modelo

O modelo nunca vê os dados de teste durante o treinamento.

---

3º - Representação

Se temos um conjunto de dados \(D\), fazemos:

\( D = D_{train} \cup D_{test} \)

\( D_{train} \cap D_{test} = \emptyset \)

Ou seja, os conjuntos são separados e não se sobrepõem.

---

4º - Proporção comum

Uma divisão típica é:

  • 70% → treino
  • 30% → teste

Outras variações comuns:

  • 80/20
  • 75/25

Não existe uma única divisão correta.

6º - Importância da aleatoriedade

A divisão deve ser feita de forma aleatória.

Sem isso, podemos introduzir viés no modelo.

Exemplo de problema:

  • dados ordenados por tempo
  • treino com dados antigos
  • teste com dados recentes

Isso pode distorcer a avaliação.

---

7º - Estratificação (classificação)

Em problemas de classificação, é importante manter a proporção das classes.

Isso é chamado de estratificação.

Exemplo:

  • 90% classe A
  • 10% classe B

Ambos os conjuntos devem preservar essa proporção.

---

8º - Limitações

  • Resultados podem variar dependendo da divisão
  • Pode não representar bem o comportamento geral

Por isso, técnicas como validação cruzada são frequentemente usadas.

---

9º - Quando usar

  • Primeira avaliação de modelos
  • Datasets grandes
  • Análises rápidas
---

10º - Conclusão

O train/test split é uma abordagem simples e essencial para avaliar modelos.

  • Evita avaliações enviesadas
  • Simula dados não vistos
  • É base para métodos mais avançados

Sem essa separação, não é possível confiar nos resultados do modelo.
