Perceptron

1º - O Problema: Classificação Linear

Considere um problema simples de classificação:

\(x_1\) \(x_2\) \(y\)
110
210
321
431

Queremos separar os dados em duas classes:

  • 0 → Classe negativa
  • 1 → Classe positiva

2º - Ideia do Perceptron

O perceptron é um modelo que aprende uma fronteira de decisão linear (uma reta que divide os dados).

Para decidir a qual classe um dado pertence, ele calcula uma combinação linear das entradas:

\( z = w_1 x_1 + w_2 x_2 + b \)

Onde cada componente tem um papel fundamental:

  • \(x_1, x_2\) (Entradas): São as características do seu dado (ex: altura e peso).
  • \(w_1, w_2\) (Pesos/Weights): Representam a importância de cada entrada. Se \(w_1\) for maior que \(w_2\), a entrada \(x_1\) tem mais influência no resultado final.
  • \(b\) (Bias/Viés): É um ajuste fixo que permite deslocar a fronteira de decisão. Sem ele, a reta de classificação seria obrigada a passar sempre pela origem (0,0).
  • \(z\) (Soma Ponderada): É o resultado bruto dessa conta. É esse valor que será enviado para a função de ativação para gerar a previsão final.

3º - Função de Ativação

O perceptron usa uma função degrau:

\( \hat{y} = \begin{cases} 1 & \text{se } z \geq 0 \\ 0 & \text{se } z < 0 \end{cases} \)

Ou seja, ele transforma o valor contínuo em uma decisão binária.

4º - Interpretação Geométrica

A equação:

\( w_1 x_1 + w_2 x_2 + b = 0 \)

define uma reta (em 2D) que separa os dados.

  • Um lado da reta → classe 1
  • Outro lado → classe 0

5º - Regra de Atualização

O perceptron aprende ajustando os pesos e bias quando erra.

Regra de atualização:

\( w_{novo} = w_{antigo} + \eta (y - \hat{y}) x \)

\( b_{novo} = b_{antigo} + \eta (y - \hat{y}) \)

Onde:

  • \(y\) é o valor real e \(\hat{y}\) é a previsão.
  • \(x\) é a entrada que causou o erro.
  • \(\eta\) (Eta) é a Taxa de Aprendizado (Learning Rate): É um número pequeno (ex: 0.1 ou 0.01) que definimos para o "tamanho do passo" no ajuste.

O papel da Taxa de Aprendizado (\(\eta\))

Imagine que o Perceptron está tentando encontrar o lugar certo para a reta de decisão:

  • Se \(\eta\) for muito alto: O modelo faz ajustes agressivos. Ele pode "pular" a posição ideal e nunca conseguir estabilizar (fica oscilando de um lado para o outro).
  • Se \(\eta\) for muito baixo: O ajuste é minúsculo. O modelo precisará de muito mais tempo e repetições para aprender a separar os dados.

Intuição do termo \( (y - \hat{y}) \)

  • Se o modelo acertou, então \(y - \hat{y} = 0\) → o ajuste é multiplicado por zero (nada muda).
  • Se o modelo errou, esse termo define a direção do ajuste (para cima ou para baixo), enquanto o \(\eta\) define a intensidade desse movimento.

Isso mostra que o Perceptron é um modelo reativo ao erro: ele só aprende quando erra, e a taxa de aprendizado controla o quão drástica é essa reação.

6º - Exemplo Numérico

Para entender o ajuste na prática, vamos supor os seguintes valores iniciais:

  • Pesos iniciais: \(w = (1, 1)\)
  • Bias inicial: \(b = -2\)
  • Taxa de aprendizado: \(\eta = 1\) (usada alta aqui para facilitar a visualização)

Considere a entrada \(x = (2, 1)\), cujo valor real esperado é \(y = 0\).

Passo 1: Calcular a Previsão

\(z = (w_1 \cdot x_1) + (w_2 \cdot x_2) + b\)
\(z = (1 \cdot 2) + (1 \cdot 1) - 2 = 1\)

Como \(z \geq 0\), a previsão da função de ativação é \(\hat{y} = 1\).
Comparando com o valor real (\(y = 0\)), o modelo errou. Precisamos atualizar os parâmetros.

Passo 2: Atualização dos Pesos e Bias

O erro é calculado como \((y - \hat{y}) = (0 - 1) = -1\).

Atualizando os Pesos:

\(w_{novo} = w_{antigo} + \eta (y - \hat{y}) x\)
\(w_{novo} = (1, 1) + 1 \cdot (-1) \cdot (2, 1)\)
\(w_{novo} = (1, 1) + (-2, -1) = (-1, 0)\)

Atualizando o Bias:

\(b_{novo} = b_{antigo} + \eta (y - \hat{y})\)
\(b_{novo} = -2 + 1 \cdot (-1) = -3\)

Após esse ajuste, com os novos valores \(w = (-1, 0)\) e \(b = -3\), o modelo agora daria um resultado negativo para a mesma entrada, corrigindo o erro anterior.

7º - De onde vêm os valores de \(w\) e \(b\) iniciais?

  • Inicialização com Zero: No Perceptron simples, é comum começar com todos os pesos e bias valendo 0. O modelo aprenderá a partir do primeiro erro.
  • Inicialização Aleatória: Atribuímos valores pequenos e aleatórios. Isso é essencial em redes neurais maiores para evitar que todos os neurônios aprendam a mesma coisa (quebra de simetria).

Independentemente do início, a regra de atualização "empurra" esses valores até encontrar a reta ideal.

9º - Limitação

O perceptron só funciona quando os dados são linearmente separáveis.

10º - Conclusão

  • É um dos primeiros modelos de redes neurais
  • Realiza classificação binária
  • Aprende uma fronteira linear
  • Atualiza pesos apenas quando erra
  • Base para redes neurais modernas
