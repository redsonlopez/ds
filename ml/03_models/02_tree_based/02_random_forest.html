No post anterior, uma única árvore encontrou uma regra perfeita.

Mas na prática, pequenas mudanças nos dados podem gerar árvores completamente diferentes.

  • A árvore é muito sensível aos dados
  • Isso é chamado de alta variância

2º - Ideia do Random Forest

Em vez de confiar em uma única árvore, usamos várias árvores ao mesmo tempo.

Cada árvore aprende uma regra um pouco diferente.

A decisão final é feita por votação (ou média).

3º - O Segredo: Criar Árvores Diferentes

O segredo do Random Forest é garantir que as árvores sejam diferentes entre si.

Isso é feito de duas formas:

  • Amostragem de Linhas (Bootstrap): cada árvore treina com um subconjunto diferente dos dados
  • Amostragem de Colunas (Feature Selection): em cada divisão, a árvore considera apenas algumas variáveis

Isso força as árvores a analisarem o problema por ângulos diferentes.

6º - Predição Final

A forma de combinar as árvores depende do tipo de problema:

Classificação (Votação)

O resultado é a classe mais frequente (moda):

\( Resultado = \text{moda} \{ T_1, T_2, ..., T_n \} \)

Regressão (Valores Numéricos)

O resultado é a média das previsões:

\( Resultado = \frac{T_1 + T_2 + ... + T_n}{n} \)

8º - Por que isso funciona?

  • Uma árvore pode errar
  • Várias árvores tendem a corrigir os erros umas das outras
  • O modelo fica mais estável
  • Reduz overfitting

9º - Conceitos Importantes

  • Bootstrap: amostragem com reposição
  • Feature Selection: uso de subconjunto de variáveis
  • Ensemble: combinação de modelos
  • Redução de variância: maior robustez

10º - Conclusão

Random Forest melhora árvores de decisão ao combinar várias delas e forçar diversidade entre elas.

Isso torna o modelo mais robusto, reduz a variância e melhora a capacidade de generalização.
