Bagging (Bootstrap Aggregating)

1º - O Problema: Modelos Instáveis

Alguns modelos, como árvores de decisão, são altamente sensíveis aos dados de treino.

Pequenas mudanças no conjunto de dados podem gerar modelos muito diferentes.

Isso resulta em alta variância, ou seja, o modelo não generaliza bem para novos dados.

---

2º - Ideia do Bagging

O Bagging tenta reduzir a variância combinando vários modelos.

Em vez de treinar um único modelo, treinamos vários modelos independentes e combinamos suas previsões.

  • Criamos várias amostras do dataset
  • Treinamos um modelo em cada amostra
  • Agregamos as previsões
---

3º - Bootstrap Sampling

O segredo do Bagging está no bootstrap.

Isso significa gerar novas amostras a partir do dataset original com reposição.

Exemplo:

Dataset original:

[1, 2, 3, 4, 5]

Amostras bootstrap:

[1, 2, 2, 4, 5]
[3, 3, 4, 5, 5]
[1, 1, 2, 3, 5]

Cada amostra tem o mesmo tamanho do dataset original, mas com repetições.

---

4º - Treinando Múltiplos Modelos

Para cada amostra bootstrap:

  • Treinamos um modelo (ex: árvore de decisão)

Se tivermos \(B\) amostras, teremos \(B\) modelos:

\( f_1(x), f_2(x), \dots, f_B(x) \)

---

5º - Agregando as Previsões

Para regressão, usamos a média:

\( \hat{y} = \frac{1}{B} \sum_{b=1}^{B} f_b(x) \)

Para classificação, usamos votação majoritária:

\( \hat{y} = \text{arg max}_{c} \sum_{b=1}^{B} I(f_b(x) = c) \)

---

6º - Intuição

Cada modelo individual pode errar de forma diferente.

Ao combinar vários modelos:

  • Os erros tendem a se cancelar
  • A previsão fica mais estável

Isso reduz a variância do modelo.

---

7º - Interpretação Geométrica

Imagine várias árvores de decisão gerando fronteiras diferentes:

Modelo 1: |---|
Modelo 2:  |----|
Modelo 3:   |---|
Modelo 4: |------|

Ao combinar todos, obtemos uma fronteira mais suave e robusta.

---

8º - Quando Funciona Bem

  • Modelos com alta variância (ex: árvores)
  • Datasets pequenos ou com ruído
---

9º - Limitações

  • Não reduz viés (bias)
  • Aumenta custo computacional
  • Modelos deixam de ser interpretáveis individualmente
---

10º - Relação com Random Forest

O Random Forest é uma extensão do Bagging aplicada a árvores de decisão.

Além do bootstrap, ele também:

  • Seleciona subconjuntos aleatórios de features

Isso aumenta ainda mais a diversidade entre os modelos.

---

11º - Conclusão

O Bagging é uma técnica simples e poderosa para reduzir variância.

Ele combina múltiplos modelos treinados em amostras diferentes do dataset.

É a base de métodos mais avançados como o Random Forest.
