Gradient Boosting

1º - O Problema: Erros dos Modelos Simples

Modelos simples, como árvores rasas, podem cometer erros sistemáticos.

Ao contrário do Random Forest, que reduz variância, aqui o foco é:

  • Reduzir o erro (bias) de forma iterativa
---

2º - Ideia do Boosting

O Boosting constrói modelos de forma sequencial.

  • Cada novo modelo tenta corrigir os erros do anterior

Em vez de árvores independentes, temos uma sequência de árvores.

---

3º - Intuição

Imagine que o modelo inicial comete erros.

O próximo modelo foca nesses erros, tentando corrigi-los.

Repetimos esse processo várias vezes.

---

4º - Modelo Aditivo

O modelo final é a soma de vários modelos simples:

\( F(x) = \sum_{m=1}^{M} \gamma_m h_m(x) \)

Onde:

  • \(h_m(x)\): modelo (geralmente uma árvore rasa)
  • \(\gamma_m\): peso do modelo
  • \(M\): número de etapas
---

5º - Ideia Central: Ajustar Resíduos

No caso de regressão com erro quadrático:

  • Começamos com uma previsão inicial
  • Calculamos os resíduos (erros)
  • Treinamos um novo modelo para prever esses resíduos

\( r_i = y_i - F(x_i) \)

O novo modelo aprende a aproximar \(r_i\).

---

6º - Gradiente Descendente

O nome "Gradient Boosting" vem da ideia de otimização.

Em vez de ajustar diretamente o modelo, seguimos o gradiente do erro.

\( r_i = - \frac{\partial L(y_i, F(x_i))}{\partial F(x_i)} \)

Ou seja, cada novo modelo aponta na direção de maior redução da perda.

---

7º - Algoritmo (Resumo)

  • Inicializar o modelo com um valor constante
  • Para cada iteração:
    • Calcular os resíduos
    • Treinar uma árvore para prever os resíduos
    • Atualizar o modelo

\( F_m(x) = F_{m-1}(x) + \eta \cdot h_m(x) \)

Onde \(\eta\) é a learning rate.

---

8º - Learning Rate

  • Pequeno: aprendizado mais lento, melhor generalização
  • Grande: aprendizado rápido, risco de overfitting

Existe um trade-off entre número de árvores e learning rate.

---

9º - Diferença para Random Forest

  • Random Forest: árvores independentes (paralelo)
  • Gradient Boosting: árvores sequenciais (dependentes)

Random Forest reduz variância.
Gradient Boosting reduz bias.

---

10º - Vantagens

  • Alta performance
  • Captura relações complexas
  • Flexível (diferentes funções de perda)
---

11º - Limitações

  • Mais sensível a overfitting
  • Treinamento mais lento
  • Requer ajuste de hiperparâmetros
---

12º - Conclusão

O Gradient Boosting constrói modelos de forma sequencial, corrigindo erros a cada etapa.

É uma das técnicas mais poderosas de machine learning, especialmente em dados tabulares.

Seu desempenho vem da combinação de muitos modelos simples ajustados de forma inteligente.
