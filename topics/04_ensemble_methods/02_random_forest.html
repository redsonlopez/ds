Random Forest

1º - O Problema: Instabilidade das Árvores

Árvores de decisão são modelos simples e interpretáveis, mas possuem um problema importante:

  • Alta variância (mudanças pequenas nos dados geram árvores muito diferentes)

Isso pode levar a overfitting.

---

2º - Ideia do Random Forest

O Random Forest resolve esse problema combinando várias árvores de decisão.

Em vez de usar uma única árvore, usamos uma floresta de árvores.

  • Cada árvore é treinada com uma amostra diferente dos dados
  • As previsões são combinadas
---

3º - Bagging (Bootstrap Aggregating)

O primeiro componente do Random Forest é o bagging.

Passos:

  • Amostramos o dataset com reposição (bootstrap)
  • Treinamos uma árvore para cada amostra
  • Repetimos várias vezes

Isso gera árvores diferentes, reduzindo a variância.

---

4º - Aleatoriedade nas Features

Além do bagging, o Random Forest introduz mais aleatoriedade:

  • Em cada divisão da árvore, apenas um subconjunto das features é considerado

Isso evita que todas as árvores fiquem muito parecidas.

---

5º - Como Funciona (Resumo)

  • Para cada árvore:
    • Amostra bootstrap dos dados
    • Cresce uma árvore de decisão
    • Em cada split, escolhe um subconjunto de variáveis
---

6º - Regra de Decisão

Classificação: votação majoritária

\( \hat{y} = \text{arg max}_{c} \sum_{t=1}^{T} I(h_t(x) = c) \)

Regressão: média das previsões

\( \hat{y} = \frac{1}{T} \sum_{t=1}^{T} h_t(x) \)

Onde:

  • \(T\): número de árvores
  • \(h_t(x)\): previsão da árvore \(t\)
---

7º - Intuição

Cada árvore individual pode cometer erros diferentes.

Ao combinar várias árvores:

  • Erros individuais tendem a se cancelar
  • O modelo final fica mais estável
---

8º - Importância das Features

O Random Forest permite estimar a importância das variáveis.

Ideia:

  • Medir o quanto cada feature reduz o erro nas árvores

Isso ajuda na interpretação do modelo.

---

9º - Vantagens

  • Reduz overfitting em relação à árvore única
  • Funciona bem sem muito ajuste
  • Lida bem com dados não lineares
  • Robusto a ruído
---

10º - Limitações

  • Menos interpretável que uma árvore única
  • Pode ser mais lento (muitas árvores)
  • Modelo maior (uso de memória)
---

11º - Relação com Bagging

O Random Forest é uma extensão do bagging:

  • Bagging: apenas amostragem dos dados
  • Random Forest: amostragem + aleatoriedade nas features
---

12º - Conclusão

O Random Forest é um dos modelos mais utilizados em machine learning.

Ele combina simplicidade com boa performance, sendo uma escolha sólida para diversos problemas.

Ao reduzir a variância das árvores, consegue generalizar melhor e evitar overfitting.
