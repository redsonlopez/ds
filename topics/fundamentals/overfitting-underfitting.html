Overfitting e Underfitting

1º - O Problema: Ajustar Bem os Dados

Ao treinar um modelo de Machine Learning, queremos que ele aprenda padrões reais dos dados.

Mas existem dois problemas comuns:

  • Underfitting: o modelo é simples demais
  • Overfitting: o modelo é complexo demais

O desafio é encontrar um equilíbrio entre esses dois extremos.

2º - Exemplo Intuitivo

Considere um conjunto de pontos que seguem uma tendência:

x y
12
24
35
44
55

Agora vamos tentar ajustar modelos diferentes.

3º - Underfitting

Um modelo muito simples pode não capturar o padrão dos dados.

Exemplo: uma linha horizontal tentando ajustar os pontos.

y = 3

Problemas:

  • Erro alto nos dados de treino
  • Não aprende a relação entre x e y

Conclusão: o modelo não é capaz de representar o problema.

4º - Overfitting

Um modelo muito complexo pode se ajustar perfeitamente aos dados de treino.

Exemplo: um polinômio de grau muito alto passando por todos os pontos.

y = ax^4 + bx^3 + cx^2 + dx + e

Problemas:

  • Erro praticamente zero no treino
  • Captura ruído (variações aleatórias ou erros nos dados) em vez de padrão
  • Péssimo desempenho em novos dados

Conclusão: o modelo "decora" os dados.

5º - Bom Ajuste (Equilíbrio)

O ideal é um modelo que capture a tendência geral sem exagerar na complexidade.

y = ax + b

Características:

  • Erro baixo (mas não zero) no treino
  • Boa capacidade de generalização

6º - Relação com o Risco Empírico

O overfitting ocorre quando minimizamos demais o risco empírico:

\( R_{emp} = \frac{1}{n} \sum L(y_i, \hat{y}_i) \)

Isso pode fazer o modelo se ajustar até ao ruído dos dados.

7º - Generalização

O objetivo real não é apenas ir bem no treino, mas em dados novos.

Chamamos isso de capacidade de generalização.

  • Underfitting: ruim no treino e no teste
  • Overfitting: bom no treino, ruim no teste
  • Equilíbrio: bom em ambos

8º - Como Identificar

Comparando erro de treino e teste:

Situação Erro Treino Erro Teste
UnderfittingAltoAlto
OverfittingBaixoAlto
Bom ajusteBaixoBaixo

9º - Como Evitar

  • Underfitting:
    • Aumentar a complexidade do modelo
    • Adicionar mais variáveis
  • Overfitting:
    • Usar mais dados
    • Aplicar regularização
    • Reduzir complexidade do modelo

10º - Conclusão

Overfitting e underfitting são dois lados do mesmo problema: o ajuste do modelo.

O objetivo do Machine Learning é encontrar um modelo que capture padrões reais,
mantendo um bom equilíbrio entre aprendizado e generalização.
