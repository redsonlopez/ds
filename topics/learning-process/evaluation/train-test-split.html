Train/Test Split

1º - O Problema: Avaliação enviesada

Se avaliarmos um modelo nos mesmos dados usados no treinamento, o erro será irrealista.

O modelo pode simplesmente se ajustar aos dados, sem realmente aprender padrões gerais.

---

2º - Ideia do Train/Test Split

Para obter uma avaliação mais confiável, dividimos os dados em dois conjuntos:

  • Treino (train): usado para ajustar o modelo
  • Teste (test): usado apenas para avaliação

O modelo não tem acesso aos dados de teste durante o treinamento.

---

3º - Formalização

\( D = D_{train} \cup D_{test} \)

\( D_{train} \cap D_{test} = \emptyset \)

Os conjuntos são separados e não se sobrepõem.

---

4º - Intuição

  • Treino → o modelo aprende
  • Teste → simula dados novos

Isso aproxima o comportamento do modelo no mundo real.

---

5º - Escolha da proporção

  • 70/30 ou 80/20 são comuns
  • Mais dados de treino → melhor aprendizado
  • Mais dados de teste → avaliação mais estável
---

6º - Boas práticas

  • Dividir de forma aleatória
  • Usar estratificação em classificação
---

7º - Limitação

O resultado depende da divisão escolhida.

Diferentes divisões podem gerar avaliações diferentes.

---

8º - Conclusão

O train/test split é a forma mais simples de avaliar modelos, mas pode gerar estimativas instáveis.
