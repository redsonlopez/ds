Engenharia de Recursos (Feature Engineering)

1º - Representação dos dados

Modelos de Machine Learning não "entendem" os dados.

Eles apenas recebem números e tentam encontrar padrões matemáticos.

O desempenho do modelo depende diretamente de como os dados são representados.

---

2º - Ideia central

Feature engineering é o processo de transformar dados brutos em uma representação mais útil.

O objetivo é facilitar o aprendizado do modelo.

---

3º - Por que isso importa?

O mesmo problema pode ser fácil ou difícil dependendo da representação.

Considere dois cenários:

  • dados mal representados → modelo não encontra padrão
  • dados bem representados → padrão se torna evidente

Ou seja, não é só o modelo que importa — a forma dos dados também importa.

---

4º - Intuição geométrica

Podemos imaginar os dados como pontos em um espaço.

O modelo tenta separar ou ajustar esses pontos.

  • dados desorganizados → difícil separar
  • dados transformados → separação mais simples

Feature engineering muda a posição dos pontos nesse espaço.

---

5º - Exemplo simples

Suponha uma relação não linear:

\( y = x^2 \)

Se usamos apenas \(x\), o modelo linear não consegue aprender bem.

Mas se criamos uma nova feature \(x^2\):

\( y = \beta_1 x + \beta_2 x^2 \)

O problema se torna mais fácil.

---

6º - Tipos de transformação

  • escalonamento → ajustar magnitudes
  • codificação → transformar categorias
  • criação de features → gerar novas variáveis
  • seleção de features → remover variáveis irrelevantes

Cada transformação altera como o modelo percebe os dados.

---

7º - Relação com o modelo

Diferentes modelos têm diferentes limitações:

  • modelos lineares → precisam de boas representações
  • modelos mais complexos → conseguem aprender mais padrões automaticamente

Mesmo assim, boas features quase sempre melhoram o desempenho.

---

8º - Risco

Transformações inadequadas podem piorar o modelo.

  • inserir ruído
  • criar relações artificiais
  • aumentar complexidade sem necessidade
---

9º - Boa prática

Feature engineering deve ser guiado por:

  • entendimento dos dados
  • conhecimento do problema
  • validação empírica

Não é um processo puramente automático.

---

10º - Conclusão

Feature engineering é a ponte entre dados brutos e aprendizado eficaz.

Uma boa representação pode ser tão importante quanto o modelo utilizado.
